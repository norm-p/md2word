### Notes:
Speaker: Norm (Solutions Architect)

We've structured this as crawl, walk, run because the goal is to scale as trust improves.

Crawl is about proving the AI works on real YETI data. We stand up a non-prod environment, train a product ID model against the highest-volume drinkware SKUs, and wire up the Salesforce sandbox. Everything is advisory — no automation, full human review, working on offline data. The goal is to answer the question: does this model actually get it right? We're extending what we showed in the POC, and if it doesn't hit the bar, we have the opportunity to tune, refine, and iterate before anything touches production. The option remains with YETI — if you'd like to turn this up in production it would still be advisory only, no autonomous write-back. But this option is geared more towards proving the AI works in your environment before operationalizing.

Walk is where we operationalize. We move to production Salesforce, expand to the full drinkware catalog plus coolers, and start running in "shadow mode" — the AI is scoring live claims but human agents are still making all the calls. That gives you a clean comparison: what the AI would have done versus what the agent did. We also layer in enhanced fraud detection through additional AI techniques. By the end of walk, the vision is that agents have a dashboard showing AI accuracy and they're actively feeding corrections back into the model to improve it over time.

Run is the long game — the future vision. Auto-approve and auto-deny on high-confidence claims — think 80/20, where the majority of cases are handled by the AI agent and exceptions still route to live agents. This is also where automated model retraining comes in, along with a customer-facing chatbot, expansion into soft goods and coolers, and integration with ERP and Salsify. To be clear, we're not recommending you start in the deep end at run. That comes after crawl or walk proves the value.

Each phase stands on its own. Crawl proves the thesis, walk doubles down, and the level of investment is gated at each step based on what the data tells you.

### Notes:
Speaker: Norm (Solutions Architect)
This is the crawl architecture. A few things to orient you on the diagram.

Container Apps is the orchestration and compute layer — intake, analysis, writeback, evaluation. This is where the business logic lives. It talks to AI Foundry for the product ID model and LLM endpoints, and Data Services for blob storage, AI Search, and Key Vault. External sources feed in as well — Confluence for the warranty SOPs and reverse image search for fraud. The Confluence integration in crawl is a lighter touch — more of an offline data export as opposed to a real-time sync — but it gives the AI the same context that agents have today. That context is where the intelligence comes from.

Important callout on the right: every case routes to human review. No autonomous approvals — nothing prematurely pushed into the live system. The pipeline writes into the Salesforce sandbox, and notes can be posted to show the logic behind each evaluation, which is valuable for tuning when the model gets it wrong and for confirming where it's hitting its marks. Agents review every recommendation, we track overrides and cost-per-claim, and we build the evidence base for moving to walk.

### Notes:
Speaker: Norm (Solutions Architect)
Same foundation as crawl, but now production-grade. A few key differences on this diagram.
The landing zone is a fuller complement for production deployment — App Gateway with a web application firewall, full API Management as a gateway on top of our APIs to enforce rate limiting, authentication policies, and traffic controls. CI/CD with release gates — deployments are pipeline-driven, not manual. These are the key things that differentiate a proving environment like crawl from a production-geared setup. AI Foundry now hosts the full product catalog with an evaluation loop for iterating and refining from a GenAI Ops and ML Ops perspective. Confluence syncs live instead of the offline export we use in crawl.
On the right, two modes. Shadow mode: AI scores live claims, agents work the case as usual, we compare outcomes — the human is making the final judgment. Then agent-assisted: AI recommendations pre-populate Salesforce production, agents confirm and provide feedback that loops back into the model. Final authority stays human.
The validation bar goes up too — we expand the evaluation set with additional production cases so the system is grounded in a larger body of real-world claims it's had the opportunity to learn from. More cases means more rigor, and the rigor matches the production stakes.
